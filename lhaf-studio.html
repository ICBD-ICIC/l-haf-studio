Attack relation

Attacks ⊆ Args × Args are not restricted by distance; only defenses are, under η- and ζ-admissibility. In L-HAF, semantically distant attacks are gradually weakened.

An attack from a distant argument loses strength proportionally to d(A,B) − ε_A once labeling is active.

Distance functions between arguments

The distance d_Ω(A,B) is induced from the hashcloud and the hashtag sets H_A, H_B. The paper presents several valid instances; the chosen one drives neighborhoods, extensions and attack weakening.

Topological metrics of the hashcloud

Local and global metrics characterizing the semantic structure, used to auto-compute the ε_A threshold of each argument.

Local: Degree centrality: deg(α) = |{β : (α,β)∈E}| Avg neighbor degree: mean{deg(β) : β ~ α} Local clustering: C(α) = triangles(α) / (deg(α)·(deg(α)−1)/2) Closeness: CC(α) = (|H|−1) / Σ d(α,β) Global: radius = min eccentricity · diameter = max eccentricity ε_A auto: ε_A = round((local + global) / 2)

Neighborhoods η — local semantics

N^ε_A = {X ∈ Args : d_Ω(A,X) ≤ ε_A}. An argument A is η-acceptable w.r.t. S if every attacker B has a defender C ∈ S ∩ N^ε_A — only semantically close arguments may defend.

Neighborhoods restrict who can defend whom.

Communities ζ — collective semantics

There is a semantic path X₁⇝Xₙ when N^ε_Xi ∩ N^ε_Xi+1 ≠ ∅ for each i. The community C_A = {B : A⇝B ∨ B⇝A}. An argument is ζ-acceptable if each attacker has a defender within its own community. Communities partition Args.

Communities are connected components in the neighborhood-overlap graph.

η / ζ extensions

Four families of extensions (grounded, preferred) under four compared semantics: classical Dung, τ-proximity (global), η-neighborhood (individual) and ζ-community (collective).

Step 2 Subarguments & aggregation

sub: A^μ → 2^(A^μ). Each argument decomposes into subarguments with disjoint hashtags that fully cover H_A. Induced attacks Attacks⁺ link subarguments with contradictory conclusions.

Conditions on sub(A^μ): i) A_i ⊆ H_A (subarg hashtags ⊆ parent) ii) ⋃ A_i = H_A (total coverage — partition) iii) A_i ∩ A_j = ∅ (strict disjunction) iv) non-trivial Strategies: singleton · bipartite · manual

Step 3 Valuation function υ

M_A is the centrality vector of A's hashtags in G_Ω. υ(A^μ) = f(M_A), where f is an aggregation operator. α_A = υ(A^μ) is the argument's initial strength.

M_A(i) = centrality of hashtag αᵢ in G_Ω Centralities m(·): degree · closeness · eigenvector · PageRank Operators f(M_A): sum (coverage) · mean · max (dominant) · min (weakest link)

Step 4 Labeling procedure ⟨α, δ⟩

Assigns ⟨α_A, δ_A⟩ to each argument. δ_A incorporates the weakening of incoming attacks, attenuated by semantic distance d(A,B) and the threshold ε_A.

If B ∈ N^ε_A (d ≤ ε_A): contribution = α_B (full strength) If B ∉ N^ε_A (d > ε_A): contribution = α_B / (d − ε_A) (weakened) If d = ∞: contribution = 0 Status: Unchallenged δ=α · Weakened 0<δ<α · Neutralized δ=0

Step 5 L-HAF extensions

Conflict-freeness, acceptability and extensions based on gradual statuses: Unchallenged (U), Weakened (W), Neutralized (N).

Strong conflict-free: S ⊆ A^υ_U Weak conflict-free: S ⊆ A^υ_U ∪ A^υ_W Strong-grounded = A^υ_U · Weak-grounded = A^υ_U ∪ A^υ_W
